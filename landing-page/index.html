Descrição do Sistema

O que é o GameVault?

O GameVault é uma aplicação web full stack acadêmica desenvolvida para gerenciamento de um catálogo de jogos. O sistema permite que o usuário visualize todos os jogos cadastrados e adicione novos títulos através de um formulário com validação de dados. A arquitetura é dividida em três camadas independentes: uma API REST construída com Flask, uma interface web desenvolvida com Flet que consome a API em tempo real, e uma landing page estática criada com HTML e Tailwind CSS para apresentação do projeto. Todos os dados são armazenados em memória (lista Python), sem uso de banco de dados.

⚙️

Backend

API REST em Flask com arquitetura de Blueprints, validação Pydantic e documentação Swagger automática.

  • ▸ GET /games — lista todos
  • ▸ GET /games/:id — detalhe
  • ▸ POST /games — cadastrar
  • ▸ Swagger em /apidocs
🎮

Frontend

Aplicação desktop em Python com Flet, tema dark gamer, consumindo a API em tempo real.

  • ▸ Lista de games com notas
  • ▸ Formulário de cadastro
  • ▸ Feedback de sucesso/erro
  • ▸ Atualização automática
🌐

Landing Page

Página estática apresentando o projeto com Tailwind CSS, design responsivo e tema visual gamer.

  • ▸ Hero section animado
  • ▸ Cards explicativos
  • ▸ Documentação da API
  • ▸ Guia de instalação

Endpoints da API

Swagger UI disponível em http://localhost:5000/apidocs

GET /games Lista todos os games 
{ "total": 5, "games": [ { "id": 1, "titulo": "The Witcher 3", ... } ] }
GET /games/<id> Retorna um game pelo ID 
{ "id": 1, "titulo": "The Witcher 3", "genero": "RPG", "plataforma": "PC", "nota": 9.8 }
POST /games Cadastra novo game (validação Pydantic)

Body (JSON):

{ "titulo": "Hades", "genero": "Roguelike", "plataforma": "PC", "nota": 9.3 }

Como Executar

Siga os passos abaixo para rodar o projeto localmente

1

Clonar / Baixar o projeto

terminal 
# Acesse a pasta do projeto
cd ldw/
2

Instalar dependências

terminal 
pip install -r requirements.txt
3

Iniciar o Backend

terminal 1 — Backend 
# No terminal 1
cd backend/
python app.py

# Saída esperada:
 * Running on http://localhost:5000
 * Swagger UI: http://localhost:5000/apidocs
4

Iniciar o Frontend

terminal 2 — Frontend 
# No terminal 2 (backend deve estar rodando)
cd frontend/
python main.py

# O browser abrirá automaticamente em:
http://localhost:8080
5

Acessar Landing Page e Swagger

Landing Page

Abrir landing-page/index.html

Duplo clique no arquivo

Swagger UI

localhost:5000/apidocs

Testar endpoints no browser
